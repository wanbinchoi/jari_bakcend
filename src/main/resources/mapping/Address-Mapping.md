# 주소 매핑 데이터 추가 가이드

## 개요
변환되지 않는 주소들을 발견했을 때, JSON 파일에 매핑 정보를 추가하는 방법입니다.

## 매핑 데이터 추가 프로세스

### 1단계: 변환 실패 주소 확인
- 서울시 API에서 받아온 원본 주소
- 예: "서울특별시 강남구 도곡동 467-12 0"

### 2단계: 실제 지도에서 검증
1. **네이버 지도**에서 주소 검색
2. **구글 지도**에서 주소 검색  
3. **카카오맵**에서 주소 검색
4. 실제 위치 확인 및 정확한 도로명주소 파악

### 3단계: JSON 파일에 추가
```json
{
  "originalAddress": "서울특별시 강남구 도곡동 467-12 0",
  "correctAddress": "서울특별시 강남구 도곡로 467",
  "roadNameAddress": "서울특별시 강남구 도곡로 467",
  "latitude": 37.4918,
  "longitude": 127.0467,
  "verificationSource": "naver_map",
  "lastVerified": "2024-10-22",
  "notes": "지번주소에서 도로명주소로 수동 변환"
}
```

## 필드 설명

### 필수 필드
- `originalAddress`: API에서 받아온 원본 주소 (정확히 일치해야 함)
- `correctAddress`: 수정된 주소
- `roadNameAddress`: 도로명주소 형태
- `latitude`: 위도 (소수점 4자리까지)
- `longitude`: 경도 (소수점 4자리까지)

### 선택 필드
- `verificationSource`: 검증한 소스 (naver_map, google_map, kakao_map, manual_verification)
- `lastVerified`: 마지막 검증 날짜 (YYYY-MM-DD)
- `notes`: 추가 설명

## 주의사항
1. **originalAddress는 서울시 API 원본과 정확히 일치**해야 합니다
2. 좌표는 **WGS84** 기준으로 입력해주세요
3. JSON 문법을 확인하고 추가해주세요
4. 파일 수정 후 `/api/admin/address/mapping/reload` API로 새로고침

## 자주 발생하는 패턴

### 패턴 1: 끝에 " 0" 붙은 지번주소
```
원본: "서울특별시 강남구 역삼동 123-45 0"
변환: "서울특별시 강남구 테헤란로 123"
```

### 패턴 2: 구주소 -> 도로명주소 변환 필요
```
원본: "서울특별시 종로구 청운동 103-1"
변환: "서울특별시 종로구 자하문로 103"
```

### 패턴 3: 상세주소 제거 필요
```
원본: "서울특별시 서초구 반포동 19-3 반포빌딩 1층"
변환: "서울특별시 서초구 반포대로 19"
```
